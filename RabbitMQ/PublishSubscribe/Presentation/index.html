What is RabbitMQ?

  • RabbitMQ is a Messaging solution that provides connections and scalablity between application components.
  • Messages are sent asynchronously between decoupled application components.
  • Each application component does not have to know about other components.
  • With messaging the receiving application component waits for incoming messages on queues.

Message Routing

  • Messages are routed through exchanges before being placed on a queue
  • A queue is bound to an exchange before receiving messages
  • A binding is a relationship between an exchange and a queue. This can be simply read as: the queue is interested in messages from this exchange.

Message Routing

Queues & Exchanges

  • A Queue stores messages that are consumed by applications.
  • Queues have to declared if does not already exist.
  • Durable queues are persisted to disk and thus survive broker restarts. Queues that are not durable are called transient.
  • An Exchange is where messages are sent.
  • Exchanges take a message and route it into zero or more queues.

Exchange Types

  • The fanout exchange is very simple. It just broadcasts all the messages it receives to all the queues it knows.
  • The direct exchange delivers message by routing key. Therefore, if a queue is bound to a direct exchange with a routing key called "A". The exchange will send messages to a queue with the routing key called "A". Messages can also be routed by using the queue name instead.
  • Topic exchanges route messages to one or many queues based on matching between a message routing key and the pattern that was used to bind a queue to an exchange. The topic exchange type is often used to implement various publish/subscribe pattern variations. Topic exchanges are commonly used for the multicast routing of messages.

What is AMQP?

  • AMQP (Advanced Message Queuing Protocol) is a networking protocol that enables conforming client applications to communicate with conforming messaging middleware brokers.
  • Enables RabbitMQ to send and receive messages via exchanges between consuming applications.
